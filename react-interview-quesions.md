Good code isn't about what the latest and what the treandiest, good code is about what writing really predictable, clean anderstandable code, and senior developers understand this.

If you find something new, learn about it, get excited about it, but be conscious that just assuming that is the best way to do something is not always the case

## Break Things Down - Trim the fat -

Although we might have more JavaScript files and more components, what we do by breaking things down that we make things easier to test, easier to understand, and less likely to break

It better to just starting something, putting something out there, start releasing products because no matter what you do, it won't be perfect.
What you wanna do is to constantly improve and the best way to improve is to get feedback on your code and feedback on your product
